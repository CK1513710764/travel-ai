# 原型设计文档 (Prototype Design Document)

## 文档信息

| 项目名称 | AI 旅行规划师 (AI Travel Planner) |
|---------|----------------------------------|
| 文档版本 | v1.0 |
| 创建日期 | 2025-11-04 |
| 设计工具 | 文本描述 + ASCII 线框图 |

---

## 1. 设计原则

### 1.1 核心设计理念

- **简洁高效**: 用户能在 3 步内完成核心任务
- **视觉优先**: 地图和图片作为主要信息载体
- **移动友好**: 响应式设计，移动端体验优先
- **反馈及时**: 所有操作都有即时视觉反馈

### 1.2 设计风格

- **色彩主题**:
  - 主色调: 蓝色系 (#1890FF) - 代表旅行和探索
  - 辅助色: 橙色 (#FA8C16) - 代表活力和热情
  - 背景色: 浅灰 (#F5F5F5)
- **字体**:
  - 标题: 思源黑体 Bold
  - 正文: 苹方 / PingFang SC
- **设计语言**: 遵循 Ant Design 设计规范

│
├─ 公开页面

#### 关键交互

1. **语音输入流程**:
   ```
   用户点击 🎤 按钮
   → 按钮变为红色，文字变为"正在听..."
   → 用户说话
   → 实时显示识别的文字

   → 用户确认和修改
   ```

   - 语音输入后自动解析并填充对应字段
   - 解析失败的部分高亮提示用户补充

3. **表单验证**:
   - 必填项未填显示红色提示
   - 日期范围不合理时提示
   - 预算过低时给出建议

## 6. 状态与反馈设计

### 6.1 加载状态

**全局加载**: 页面级别加载使用全屏 Spin 组件

**局部加载**: 按钮加载使用 Loading 图标 + 文字

**骨架屏**: 列表加载时使用 Skeleton 占位

### 6.2 空状态

每个列表页面都需要设计空状态:
- 图标 + 文案
- 引导性CTA按钮

示例:
```
🗺️
还没有旅行计划

开始创建你的第一次旅行吧！

[+ 创建行程]
```

### 6.3 错误处理

**表单验证错误**: 输入框下方显示红色提示

**API 错误**: Toast 通知 + 错误详情

**网络错误**: 全局提示 "网络连接失败，请检查网络"

**404 页面**: 友好的 404 页面 + 返回首页按钮

### 6.4 成功反馈

**操作成功**: Toast 通知 "保存成功"

**创建成功**: 跳转到详情页 + 成功提示

**删除成功**: Toast + 列表自动刷新

## 7. 动画与过渡

### 7.1 页面过渡

- 路由切换: 淡入淡出 (300ms)
- 模态框弹出: 缩放 + 淡入 (200ms)

### 7.2 列表动画

- 列表项进入: 从下往上滑入
- 列表项删除: 淡出 + 高度折叠

### 7.3 交互反馈

- 按钮点击: Scale 微缩 (100ms)
- 卡片悬停: 阴影加深 + 轻微上浮

## 8. 可访问性设计 (Accessibility)

### 8.1 键盘导航

- 所有交互元素支持 Tab 键导航
- 表单支持 Enter 键提交

### 8.2 颜色对比

- 文字与背景对比度 > 4.5:1
- 重要按钮对比度 > 7:1

### 8.3 屏幕阅读器

- 图片添加 alt 文本
- 按钮添加 aria-label
- 表单添加 label 关联
